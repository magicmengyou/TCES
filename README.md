# 培训中心电子存档管理系统

这是一个用于培训中心的电子存档管理系统，支持多种云存储服务和数据库配置，完全兼容GitHub Pages部署。

## 功能特点

- 用户身份验证和权限管理
  - 管理员：可以设置存档位置和管理所有内容
  - 教师：可以管理学生信息和课程内容
  - 学生：只能访问个人基本信息和课程资源
- 浏览课程档案和相关学习材料
- 支持多种云存储服务：
  - NAS存储
  - 百度网盘
  - 迅雷网盘
  - 夸克网盘
- 支持多种数据库存储用户账号信息：
  - Firebase
  - Supabase
  - IndexedDB (本地存储)
- 可配置单一存储或同时存储多份数据
- 响应式设计，支持移动设备
- 数据持久化存储在浏览器本地

## 技术栈

- HTML5
- CSS3
- JavaScript (原生，无框架依赖)
- localStorage (用于配置存储和用户会话管理)

## 如何部署到GitHub Pages

1. 将此仓库推送到GitHub
2. 在仓库设置中找到"Pages"选项
3. 在"Source"部分选择"Deploy from a branch"
4. 选择主分支(main)并保存
5. 等待几分钟后，您的网站将在 `https://<username>.github.io/<repository-name>/` 上线

## 使用说明

1. 打开 `login.html` 文件访问登录页面
2. 使用以下默认账户登录：
   - 管理员: admin / admin123
   - 教师: teacher / teacher123
   - 学生: student / student123
3. 登录后根据您的角色访问相应功能：
   - 管理员可以配置存储位置和管理所有内容
   - 教师可以管理学生信息和课程内容
   - 学生只能查看个人信息和课程资源
4. 使用顶部导航栏切换不同模块
5. 在"存储配置"中配置您要使用的云存储服务（仅管理员可见）
6. 在"数据库配置"中选择和配置数据库（仅管理员可见）
7. 在管理面板中可以添加新的课程档案（教师和管理员可见）
8. 使用搜索框可以快速查找档案信息

## 注意事项

由于GitHub Pages是静态网站托管服务，本系统使用浏览器的localStorage来存储配置信息。在生产环境中，建议使用后端服务来存储数据。
